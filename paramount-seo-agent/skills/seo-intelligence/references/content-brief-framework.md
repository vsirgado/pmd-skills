# Content Brief Framework — Complete Generation System

## What Makes a Great Content Brief

A content brief is a complete blueprint that a writer (human or AI) can use to produce a page that ranks. It must include: target keyword, search intent, audience, structure, competitive insights, required elements, and success criteria.

---

## Content Brief Template

### SECTION 1: OVERVIEW
```
CLIENT: [Practice name]
URL TARGET: [Proposed URL slug]
PAGE TYPE: [Service page / Blog post / Condition page / Location page / FAQ]
PRIORITY: [High / Medium / Low]
AUTHOR: [Physician name + credentials — must appear as byline]
DEADLINE: [Target publish date]
```

### SECTION 2: KEYWORD TARGETING
```
PRIMARY KEYWORD: [exact match keyword]
Monthly Search Volume: [X searches/mo]
Keyword Difficulty: [Easy / Medium / Hard — 0-100]
Current Ranking: [Position X or "Not ranking"]
Target Ranking: [Page 1, Position 1-3]

SECONDARY KEYWORDS (include naturally):
- [keyword 2] (X/mo)
- [keyword 3] (X/mo)
- [keyword 4] (X/mo)

LSI / SEMANTIC KEYWORDS (weave throughout):
- [related term 1]
- [related term 2]
- [related term 3]

LONG-TAIL VARIATIONS (great for FAQ section):
- [long-tail 1]
- [long-tail 2]
- [long-tail 3]
```

### SECTION 3: SEARCH INTENT ANALYSIS
```
INTENT TYPE: [Informational / Commercial / Transactional / Navigational]

WHAT THE SEARCHER WANTS:
[2-3 sentences describing what someone searching this keyword actually wants to find, do, or know]

STAGE IN PATIENT JOURNEY:
[ ] Awareness — learning about a condition or symptom
[ ] Consideration — comparing treatment options
[ ] Decision — ready to book an appointment
[ ] Post-treatment — looking for recovery info

EMOTIONAL CONTEXT:
[What is the searcher feeling? Fear, pain, curiosity, urgency? This shapes tone.]
```

### SECTION 4: COMPETITIVE ANALYSIS
```
TOP 3 COMPETING URLS:
1. [URL] — Word count: X | Notable: [what makes it rank]
2. [URL] — Word count: X | Notable: [what makes it rank]
3. [URL] — Word count: X | Notable: [what makes it rank]

CONTENT GAPS (what they don't cover that we should):
- [Gap 1]
- [Gap 2]
- [Gap 3]

OUR DIFFERENTIATOR:
[Why our version will be better / more authoritative / more helpful]
```

### SECTION 5: PAGE STRUCTURE
```
TITLE TAG (50-60 chars):
[exact title tag text]

META DESCRIPTION (150-160 chars):
[exact meta description text]

H1 (contains primary keyword):
[H1 text]

RECOMMENDED WORD COUNT: [X-Y words]

OUTLINE:

H2: [Introduction subheading OR direct answer section]
  - Cover: [what this section must address]
  - Include: [specific info, stats, or claims required]

H2: [Core topic section]
  - Cover: [what this section must address]
  H3: [Subsection]
  H3: [Subsection]

H2: [Benefits / How It Works / Who Is a Candidate / etc.]
  - Cover: [specifics]
  H3: [Subsection if needed]

H2: What to Expect During [Treatment/Procedure]
  - Step-by-step format
  - Numbered list for AI Overviews optimization

H2: [Risks / Recovery / Cost / Insurance — as relevant]

H2: [Comparison section if applicable — "X vs Y"]

H2: Frequently Asked Questions About [Primary Keyword]
  H3: [Question 1 — from "People Also Ask"]
  H3: [Question 2]
  H3: [Question 3]
  H3: [Question 4]
  H3: [Question 5]

H2: Schedule Your [Service] Consultation in [City]
  - CTA paragraph
  - Contact info / booking link
```

### SECTION 6: REQUIRED ELEMENTS
```
MUST INCLUDE:
- [ ] Author byline: Dr. [Name], [Specialty], [Credentials]
- [ ] Author bio box with photo (links to full bio page)
- [ ] Date published + date last reviewed
- [ ] [X] internal links (specify target pages)
- [ ] [X] external citations (list acceptable sources)
- [ ] Call-to-action: [specific CTA text and link]
- [ ] Primary image: [description + alt text]
- [ ] [Additional images/diagrams if relevant]

SCHEMA TO IMPLEMENT:
- [ ] Article or MedicalWebPage
- [ ] FAQPage (for FAQ section)
- [ ] Physician (for author)
- [ ] BreadcrumbList

COMPLIANCE NOTES:
- [ ] Include medical disclaimer: "This content is for informational purposes only and is not a substitute for professional medical advice."
- [ ] Do not make specific outcome guarantees
- [ ] Before/after references must comply with state medical board advertising guidelines
```

### SECTION 7: AI SEO REQUIREMENTS
```
DIRECT ANSWER PARAGRAPH (first 60 words of body content):
[Draft the direct answer paragraph here — this is what AI will cite]

FAQ QUESTIONS FOR AI OVERVIEWS (answer each in 2-4 sentences):
1. [Question]
2. [Question]
3. [Question]

KEY STATISTICS TO INCLUDE (with sources):
- [Statistic 1] — Source: [Source, Year]
- [Statistic 2] — Source: [Source, Year]

ENTITY SIGNALS:
- Physician name appears as: [exact format]

```

### SECTION 8: SUCCESS CRITERIA
```
90-DAY TARGETS:
- Target position: [Page 1, Position X]
- Target organic traffic: [X visits/mo]
- Target conversions: [X form fills or calls/mo]

MEASUREMENT:
- Google Search Console: track impressions + clicks for primary keyword
- Google Analytics: track organic sessions to this URL
- GHL: track leads attributed to organic source
```

## Content Brief Generation Process

When generating a content brief, execute these steps in order:

1. **Search** for the primary keyword to analyze current SERP
2. **Identify** the top 3 ranking URLs and fetch them to analyze structure
3. **Extract** People Also Ask questions (these become FAQ questions)
4. **Identify** Related Searches (these become secondary keywords)
5. **Analyze** word counts and content gaps in top results
6. **Draft** the complete brief using the template above
7. **Push** the completed brief to Notion as a new page in the client's content calendar
